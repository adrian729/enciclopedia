## 1. Supercharge the defaults

- **You don't always need new elements for flare** — a lot of polish comes from "supercharging" what's already there instead of adding more UI.
- **Replace bulleted list markers with icons** — checkmarks and arrows are safe generic choices; content-specific icons (e.g., a padlock for security features) add even more meaning.
- **Promote testimonial quotes into visual elements** — increasing the quote-mark size and changing its color turns a plain block of text into something deliberate.
- **Style links beyond the browser default** — a simple color and font-weight change works, or go further with a thick, colorful custom underline that partially overlaps the text.
- **Customize checkboxes and radio buttons** — swapping the browser default selected state for one of your brand colors is often all it takes to move a form from boring to polished.

## 2. Add color with accent borders

- **Non-designers can still add visual flair** — colorful accent borders are a low-effort substitute for photography or illustration when a UI feels bland.
- **Places accent borders work** — across the top of a card, under an active nav item, along the side of an alert, as a short accent beneath a headline, or across the top of the entire layout.
- **It's just a colored rectangle** — no graphic-design talent needed, but it goes a long way toward making something feel "designed."

## 3. Decorate your backgrounds

Even with good hierarchy, spacing, and typography, a design can feel plain. Three ways to break the monotony without overhauling the layout:

| Approach | How to apply it |
|---|---|
| **Change the background color** | Emphasize an individual panel or distinguish entire page sections. For a more energetic look, use a slight gradient — keep the two hues no more than about 30° apart |
| **Use a repeating pattern** | Subtle repeatable patterns (e.g., Hero Patterns) work across the whole background or along a single edge. Keep contrast between background and pattern low so it doesn't fight the content |
| **Add a simple shape or illustration** | A single geometric shape, a small chunk of a repeatable pattern, or something richer like a simplified world map. Same rule — low contrast so it doesn't interfere with content |

## 4. Don't overlook empty states

- **Empty states are not an afterthought** — if a feature depends on user-generated content, the empty state is the user's first interaction with it, and a blank screen kills excitement for a new feature you spent weeks designing.
- **Use an image or illustration plus a clear call-to-action** — grab attention and push the user toward the next step rather than leaving them staring at an empty table.
- **Hide supporting UI that has nothing to act on** — tabs, filters, and other controls for zero content just add noise; keep them out until there's content to operate on.
- **Treat empty states as an opportunity** — they're a chance to be interesting and exciting, not plain and boring.

## 5. Use fewer borders

Borders create separation, but too many make a design feel busy and cluttered. Reach for these alternatives first:

| Alternative | When it works |
|---|---|
| **Box shadow** | Outlines an element more subtly than a border. Works best when the element's background color differs from the page background |
| **Two different background colors** | Slightly different backgrounds on adjacent elements are usually enough to distinguish them. If you're already using different backgrounds *and* a border, try dropping the border |
| **Extra spacing** | The most direct form of separation — increase the gap and you create distinction between groups without adding any new UI |

## 6. Think outside the box

- **Preconceived component shapes are not rules** — a "dropdown" doesn't have to be a white box with a drop shadow and a stack of links; it's just a floating box and can contain sections, multiple columns, supporting text, or icons.
- **Tables don't have to be one-column-per-datum** — if a column doesn't need to be sortable, combine it with a related column to introduce hierarchy; table cells can hold images or color, not just plain text.
- **Radio buttons can be selectable cards** — when a radio group is a central part of the UI, a stack of labels with little circles is the least interesting option available.
- **Constraints are powerful, but so is freedom** — don't let habit about "how a component is supposed to look" cap the quality of the design.
